[비즈니스 로직의 수정 및 추가]
feat - 새로운 기능 추가 (비지니스 로직 생성과 생성된 모듈의 책임 확장 등)
fix - 버그 관련 수정 (버그 및 비지니스 로직이 제공하는 요소에 변동이 생기는 수정 등)
ci - CI관련 설정 수정 (git actions 등)
chore - 기타 변경사항 (빌드 스크립트 수정, 비즈니스 로직이 제공하는 요소변동이 없는 정도의 수정 등)

[비즈니스 로직의 수정이 없는 경우]
style - 스타일 (코드 형식, 세미콜론 추가, 간격: 비즈니스 로직에 변경 없는 경우)
refactor - 코드 리팩토링 (내부 비지니스 로직의 책임 분리와 효율성 개선 및 단순화 등)
docs - 문서 (문서 및 주석 추가, 수정, 삭제)
test - 테스트 (테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없는 경우)

[빌드 및 환경요소 과련 수정]
build - 빌드 관련 파일 수정(server 생성 요소 및 초기화 관련 코드 등)


ex)
git commit -m 'feat: 퀘스트에 대한 댓글 정보 제공 기능 추가'
